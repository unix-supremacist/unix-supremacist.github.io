Operating Systems

Alpine Linux

A lot like arch linux, uses busybox, musl, and openrc by default extremely lightweight great for low resource machines. also has a installer a lot like openbsds, really a great mix between arch and openbsd.

OpenBSD

A lot like FreeBSD but with more of a effort towards correct code, security on servers, and the Unix philosphy, it's extremely hard to get working on a desktop and drivers are lacking, but if it works go for it, it's atleast as good as FreeBSD in most cases

Artix

A fork ofoOne of the most versatile linux distributions around. it doesn't have a graphical user enviroment by default, however it is easy to install one, and it being a barebones system allows you to customize it way beyond most other distros. not to mention, the AUR allows you to find basically anything you can dream of in terms of software. it has a more difficult learning curve, but is well rounded, without systemd, it also has releases with desktop enviroments preinstalled.

Void linux

Another barebones and minimal linux distribution like Arch. there is a musl version which is way lighter weight than glibc, also uses the runit init system which is way faster.

9Front

A Utilitarian operating system from the same group who created Unix, in a lot of ways it is essentially Unix2, it focuses on simplicity and reusablity like original Unix but pushes it into a graphical era. forked later with additional development including but not limited to more hardware support, nes/snes/gb/gbc/md emulators, the Constitution of the United States, The Manifesto of the Commmunist Party, and a whole lot more, just generally the updated version of Plan9, a lot of the dev's worked on the original aswell.

If i made my own os/distro

Simple. Correct. Extensible. And Choice Driven

Write up on how i think operating systems should be designed, sort of tired of the current systems we have.

Simple. a operating system should be simple, a low line count auditable codebase, but also low memory usage.

Correct. do it right, or not at all, do not half ass implement bluetooth drivers to drive the userbase up.

Portable. every part of the OS should be a tool that can be used in conjunction with others, portable to other systems and usable with other software ecosystems.

Choice Driven. the operating system should be respectful of the users choice, and everything should be changible by an adept user.
